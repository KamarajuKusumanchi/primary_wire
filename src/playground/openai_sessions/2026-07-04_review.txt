Score them.

# Architecture risk I noticed

Your current model assumes:

```text
Company
   → Platform
      → Scraper
```

That's good.

The long-term risk is adding company-specific exceptions inside platform scrapers.

Avoid patterns like:

```python
if slug == "xyz":
    ...
```

or

```python
if slug == "abc":
    special selector
```

Once those accumulate, scraper maintenance becomes difficult.

Prefer:

```yaml
source:
  overrides:
      listing_selector: ...
```

or

```python
platform plugin + source config
```

so platform code stays generic.
