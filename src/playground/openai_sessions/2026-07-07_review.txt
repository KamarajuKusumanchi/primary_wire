# What I'd improve

## 1. Introduce a BaseScraper class

Right now every scraper still feels fairly standalone.

I'd introduce something like

```python
BaseScraper
    fetch()
    parse()
    normalize()
    write()
```

Then

```
Q4Scraper(BaseScraper)

InvestorRoomScraper(BaseScraper)

NotifiedScraper(BaseScraper)
```

The orchestration becomes much cleaner.

---

## 2. Standardize scraper outputs

Every scraper should return exactly the same object.

For example

```python
PressRelease(
    title,
    url,
    publish_date,
    publish_time,
    source,
    ticker,
    platform
)
```

instead of dictionaries.

That makes downstream tooling much easier.

---

## 3. Reduce HTML parsing duplication

I noticed each scraper still contains quite a bit of BeautifulSoup logic.

Many pages ultimately boil down to

```
find article

extract title

extract date

extract URL
```

A shared parser helper layer could remove another few hundred lines.

---

## 4. More tests

This is probably the biggest engineering gap.

Scrapers inevitably break.

I'd snapshot representative HTML pages.

Example:

```
tests/

q4/

page1.html

page2.html

page3.html
```

Then

```
pytest
```

can tell you immediately if a selector change broke extraction.

Without snapshots, regressions are hard to catch.

# Long-term vision

I think the project is gradually becoming less of a "collection of scrapers" and more of an **official press release ingestion framework**. To reinforce that direction, I'd organize future work around:

1. **Coverage expansion**: add support for more IR platforms to maximize the number of companies covered per new scraper.

3. **Automation**: automatically discover IR URLs, detect platforms, and generate candidate configuration entries.

5. **Observability**: dashboards for coverage, scrape success rates, stale sources, and newly detected unsupported platforms.
