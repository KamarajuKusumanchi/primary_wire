====== primary_wire: Tasks ======

This project uses [[https://www.pyinvoke.org/|Invoke]] as a small task runner
for repetitive maintenance commands -- currently, regenerating the files
under ''reports/latest/''.

===== Why Invoke =====

Previously, the three reports were regenerated by hand with:

  python.exe src/detect_ir_platform.py      > reports/latest/ir_platform.txt
  python.exe src/missing_tickers.py         > reports/latest/missing_tickers.txt
  python.exe src/check_scraper_coverage.py  > reports/latest/scraper_coverage.txt

That works, but it's easy to typo a path, forget one of the three commands,
or forget to run it before committing. Invoke replaces this with a single
named command (''invoke reports''), documents itself (''invoke --list''),
and is plain Python underneath -- no new syntax to learn, and it runs the
same way on Windows, Mac, and Linux (a plain Makefile would not, since
''make'' is not available out of the box on Windows).

===== Setup =====

Invoke is installed as part of the ''dev'' extra:

  uv sync --extra dev

===== Usage =====

List available tasks:

  invoke --list

Regenerate all three reports (this is also the default task, so plain
''invoke'' with no arguments does the same thing):

  invoke reports

Regenerate just one report:

  invoke ir-platform
  invoke missing-tickers
  invoke scraper-coverage

Each task runs its script via ''uv run python <script>'' (using an absolute
path to the script), captures stdout, and writes it to the corresponding
file under ''reports/latest/''. This works whether or not a virtual
environment is currently activated, and from any directory inside the repo
-- see "Running from other directories" below.

===== Running from other directories =====

Invoke searches upward from your current directory for ''tasks.py'', the
same way ''git'' searches upward for ''.git''. So ''invoke reports'' works
from the repo root or from any subdirectory inside the repo (e.g. ''src/'',
''data/2026/''), not just the root. It will *not* find ''tasks.py'' if you
run it from outside the repo entirely.

Script and output paths in ''tasks.py'' are resolved as absolute paths
relative to ''tasks.py'' itself (not relative to your current directory), so
regenerating reports gives the same result no matter where inside the repo
you run it from.

===== Adding new tasks =====

Tasks live in ''tasks.py'' at the repo root. Add a new ''@task''-decorated
function there, following the existing ones as a template.
